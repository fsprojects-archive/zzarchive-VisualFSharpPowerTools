Union pattern match case generation

The feature is activated via a smart tagger on a union type or a union case constructor, inside a match x with or function Case1 | Case2 expression. When the cursor is placed on one of these identifiers, the smart tagger will suggest users to automatically generate pattern match clauses.

Note that if the current line is match x or function, the code might not be parseable (i.e. no semantic highlighting for the current line). We recommend users to generate pattern match cases while the code is parsable e.g. after writing match x with Case1 -> () or function Case1 -> ().
